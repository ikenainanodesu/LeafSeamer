# LeafSeamer 项目现状报告重排设计

## 目标

将现有项目现状报告从“主题章节 + 增量补充”的组织方式，调整为以 Bundle 为主线的问题台账。读者应能快速定位某个 Bundle，理解其功能与独立工作条件，并逐项看到问题证据、风险和可执行的改善建议。

## 范围

- 保留已有项目功能、技术实现、架构、构建 workflow、工具链、安全分析和改善建议。
- 合并当前重复出现的“潜在安全问题”“建议改善”“增量研究”和专项方案内容。
- 不修改 LeafSeamer 业务代码，不重新执行源码审计，不新增未经源码或现有报告支持的结论。
- 源码清单保留为折叠附录。

## 信息架构

报告按以下顺序组织：

1. 报告摘要：日期、分支、提交、审计范围和关键结论。
2. 项目概览：功能、语言、架构、构建方式和主要工具。
3. Bundle 状态总览：Bundle、功能、依赖、独立运行条件、最高风险和问题数量。
4. 平台共享问题：部署边界、身份与权限、Secret、备份、消息校验、CI 和构建隔离。
5. Bundle 档案：按 Bundle 分组展示功能、依赖和问题。
6. 分阶段整改路线：汇总 P0、P1、P2 的落地顺序，避免重复问题正文。
7. 附录：审计范围、限制和源码清单。

Bundle 顺序为：

- Platform / Shared Concerns
- Seamer
- Logger System
- Schedule Manager
- VB Matrix Control
- OBS Control
- ATEM Control
- Mixer Control
- Graphics Package
- Backup System
- Data Sync Service
- Adapters

跨 Bundle 问题只在 Platform / Shared Concerns 中完整说明。各 Bundle 通过“关联平台问题”链接引用，不复制正文。

## Bundle 档案结构

每个 Bundle 章节包含：

- 功能定位
- 独立运行状态
- 必需依赖与可选集成
- 数据与命令入口
- 已确认问题数量和最高优先级
- 问题卡片列表

Adapters 作为一类集中展示，并明确它们不是独立业务核心，而是仅在两侧 Bundle 同时存在时启用的可选桥接层。

## 问题卡片规范

每个问题使用唯一编号，例如 `PLAT-01`、`LOGGER-01`、`VB-01`。卡片固定包含：

- 问题：一句话描述。
- 优先级：P0、P1 或 P2。
- 现状证据：现有源码、配置或报告中的事实依据。
- 风险：可能发生什么以及严重性原因。
- 影响范围：涉及的 Bundle、数据、设备或部署环境。
- 短期缓解：可以低成本立即采取的措施。
- 长期改善：推荐的目标方案。
- 实施路径：按顺序列出落地步骤。
- 验收标准：可观察、可复测的完成条件。
- 关联问题：必要时链接到平台共享问题或其他 Bundle 问题。

问题卡片默认展开，长实施路径可使用原生 `details` 折叠，打印时全部展开。

## 视觉与交互

- 使用浅色正文与深色固定侧栏，形成技术审计文档风格。
- 桌面端采用侧栏目录 + 主内容双栏布局；侧栏显示 Bundle 名称和问题计数。
- 窄屏切换为顶部目录，正文保持单栏，不产生横向滚动。
- P0、P1、P2 使用红、琥珀、蓝灰三种颜色，同时保留文字标签，避免仅靠颜色传递信息。
- 表格只用于紧凑总览；详细解释使用无嵌套的问题卡片。
- 提供“仅看 P0 / P1 / P2”和“展开全部 / 折叠实施细节”的轻量筛选控件；禁用 JavaScript 时所有内容仍可阅读。
- 使用锚点链接和 `scroll-margin-top`，确保目录跳转后标题不被遮挡。
- 打印样式隐藏目录和筛选控件，移除阴影，并保留风险标签。

## 内容迁移规则

- “项目功能”“项目实现”压缩为概览，不重复解释问题。
- 原“潜在安全问题”和“建议改善”按问题合并，不再分别维护两张列表。
- 原“部署与硬件安全方案”“敏感数据剥离与访问码管理”“前后端隔离与 Schema 校验”归入平台共享问题。
- 原“VB Matrix 扫描降压方案”归入 VB Matrix Control。
- 原“Bundle 数据回调与数据库评估”拆分到平台共享、Logger、OBS、Data Sync 和 VB Matrix 的相关问题中；数据库结论保留在平台共享的数据治理问题内。
- 不具备明确证据的建议标记为“建议验证”，不写成已确认缺陷。

## 验收标准

- 每个核心 Bundle 都有独立章节，能在目录中直接访问。
- 每个问题都同时包含风险与改善建议，不需要跨章节拼接信息。
- 同一问题只保留一份完整正文，其他位置使用引用链接。
- 桌面和窄屏均无横向溢出，长路径和代码文本可换行。
- 页面在禁用 JavaScript 时仍完整可读。
- 所有目录锚点有效，问题编号唯一。
- HTML 通过基本结构检查，且没有未闭合标签。
- 目标文件最终覆盖到 `D:\git-repo\PROFILE-MANAGER-1\nodecg\LeafSeamer_Project_Status_Report.html`。

## 验证方式

- 使用静态脚本检查重复 ID、锚点目标、问题编号和必填字段。
- 检查桌面与窄屏 CSS 断点和横向溢出。
- 由于应用内浏览器拒绝自动化访问 `file://` 页面，浏览器视觉确认由用户在现有标签页刷新后完成；自动验证仅覆盖静态 HTML、CSS 和结构规则。
